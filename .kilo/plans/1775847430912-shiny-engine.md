## Plano 2: Passar callback_url para o worker do agente

### Problema
O `callback_url` enviado no payload de entrada do Disparador não está sendo repassado para o worker do agente (backend). O agente espera receber o `callback_url` para enviar o callback de resposta, mas está chegando vazio.

### Análise
Em `disparador/app/services/dispatcher_engine.py:68-82`, o `agent_payload` é construído mas não inclui o `callback_url` que vem do payload de entrada:

```python
# agent_payload atual
agent_payload = {
    "message": "DISPARADOR_START",
    "session_id": f"{contact['number']}_{service_id}",
    "agent_id": str(config.agent_id) if config.agent_id else None,
    "context_data": {...},
    "transition_data": transition_data,
}
# FALTA: "callback_url": callback_url
```

### Solução
Adicionar o `callback_url` ao `agent_payload` antes de enviar para o agente:

```python
agent_payload = {
    "message": "DISPARADOR_START",
    "session_id": f"{contact['number']}_{service_id}",
    "agent_id": str(config.agent_id) if config.agent_id else None,
    "context_data": {...},
    "transition_data": transition_data,
    "callback_url": callback_url,  # ADICIONAR
}
```
