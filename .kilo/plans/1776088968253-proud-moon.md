# Plano: Corrigir e Melhorar Thinker

## Problemas Reportados:

1. **Agente Think Celulas é Disparado do nada**: O usuário criou um agente Thinker ("Agente Think Celulas") e ele está sendo chamado automaticamente pelo Orquestrador, sem necessidade de keyword. Provavelmente está na lista de collaborators.

2. **Keywords não funcionam**: Mesmo com keywords no Orquestrador 1, Orquestrador 2 e Thinker, o Thinker não ativa

3. **Thinker não tem acesso às tools do host**: Não consegue ver collaborators, MCPs, skills do Orquestrador 2

---

## Resposta do Usuário:

1. **Agente Disparado**: É "Agente Think Celulas" - um agente Thinker criado pelo usuário
2. **Modelo**: Quer mostrar todos os providers (OpenAI, OpenRouter, custom)
3. **Keywords**: Devem ficar na ABA DO THINKER, não no agente principal

---

## Solução Proposta:

### 1. UI - Modificar Thinker como "ferramenta" do agente

Na aba Thinker, adicionar:

```vue
<!-- Thinker Sempre Ativo -->
<v-switch
  v-model="formData.thinker_always_active"
  label="Thinker Sempre Ativo"
  hint="Ativa o Thinker em todas as mensagens, sem necessidade de keyword"
/>

<!-- Keywords do Thinker (só aparece se sempre ativo estiver OFF) -->
<v-combobox
  v-model="formData.thinker_keywords"
  label="Palavras-Chave do Thinker"
  hint="Thinker será chamado quando a mensagem contiver estas palavras"
  multiple
  chips
/>

<!-- Modelo com todos os providers -->
<v-autocomplete
  v-model="formData.thinker_model"
  :items="allModelsFromAllProviders"
  label="Modelo do Thinker"
/>
```

### 2. Backend - Campos necessários:

- `thinker_always_active` (Boolean) - ativa sem keyword
- `thinker_keywords` (List[str]) - keywords próprias do Thinker

### 3. Backend - Corrigir ativação:

Em `thinker_service.py`:
- Usar `thinker.thinker_keywords` ou `thinker.trigger_keywords` para detectar ativação
- Se `thinker_always_active = true`, sempre ativar
- Se não, verificar keywords

### 4. Backend - Injetar tools do host:

Em `thinker_service.py` → `call_thinker`:
- Buscar o agente host (onde o Thinker está configurado)
- Carregar suas tools (collaborators, MCPs)
- Passar para o Thinker usar no planejamento

### 5. Remover problema do "agente do nada":

O problema é que o Thinker está como collaborator do Orquestrador. O Orquestrador chama todos os collaborators automaticamente.

**Solução**: 
- Thinker NÃO deve ser adicionado como collaborator
- Thinker deve ser chamado APENAS via sistema de keywords/sempre ativo
- Remover lógica que chama Thinker como collaborator

## Pergunta para o usuário:

Sobre o "Agente Think Celulas" ser Disparado do nada:
- **CONFIRMADO**: Ele está como **collaborator** do Orquestrador, por isso é chamado automaticamente
- O Orquestrador chama todos os collaborators sem verificar se é Thinker ou não
- Por isso ele é Disparado do nada - não é pela keyword, é porque está na lista de collaborators!

**Solução**: Remover o Thinker da lista de collaborators. O Thinker deve ser chamado APENAS via:
1. Keywords próprias na aba Thinker, ou
2. "Thinker Sempre Ativo"
