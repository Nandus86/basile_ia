# Plano: Ativar Thinker em todos os agentes colaboradores

## Problema:
O Thinker está funcionando apenas no primeiro agente orquestrador (`Sistema - General Orch`), mas não nos agentes colaboradores subsequentes que são chamados durante a orchestration (como `BasileIA - Agente de Células - Orquestrador`).

O log mostra:
```
[Task] 🧠 Thinker activated by keyword: 'presença'  ← No primeiro agente
[Task] 🔍 DEBUG Thinker - checking keywords: ... in message: olá...  ← No segundo NÃO aparece
```

## Análise:
A lógica do Thinker está implementada apenas em `tasks.py` na função principal `process_message_task`. 

Quando um agente orquestrador chama colaboradores através de `_invoke_collaborator` em `agent_orchestrator.py`, essa função:
1. Cria o `agent_config` via `factory.get_agent_config()`
2. Modifica o `system_prompt` com instruções de hierarquia
3. Executa o agente

Mas **NÃO** verifica se o Thinker deve ser ativado para esses colaboradores.

## Solução:
Adicionar a lógica do Thinker em `agent_orchestrator.py` no método `_invoke_collaborator`, logo após a linha onde o system_prompt é modificado (~linha 243), para que todos os agentes colaboradores também verifiquem e ativem o Thinker quando apropriado.
