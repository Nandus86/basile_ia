# Plano: Corrigir Thinker - Lista de Tasks, Modelo e Redis

## Problema:
O usuário reportou 3 problemas com a implementação atual do Thinker:

1. **O Thinker ativa o mesmo modelo para todos os agentes abaixo** - Quando o Thinker é ativado num agente, o código substitui o modelo do agente principal pelo modelo do Thinker. Isso está errado - cada agente deve usar seu próprio modelo.

2. **O JSON não está sendo retornado** - O Thinker service não está retornando a lista de tarefas (passos). O modelo pode não estar gerando o formato JSON esperado.

3. **Não está salvando no Redis** - A memória não está sendo salva corretamente.

## Análise dos Problemas:

### Problema 1: Modelo errado sendo aplicado
**Local:** `tasks.py` linhas ~1795-1798 e `agent_orchestrator.py` linhas ~376-379

O código atual:
```python
thinker_model = getattr(agent_model, 'thinker_model', None)
if thinker_model:
    agent_config["model"] = thinker_model
```

Isso substitui o modelo do agente principal pelo modelo do Thinker, afetando todos os colaboradores.

**Correção:** O modelo do Thinker (thinker_model) deve ser usado APENAS na chamada `call_thinker` para gerar o plano. O agente principal deve continuar usando seu próprio modelo. Remover a lógica de override do modelo do agente principal.

### Problema 2: JSON não retornado
**Local:** `thinker_service.py` e `tasks.py`

O Thinker pode não estar gerando o JSON corretamente. Possible reasons:
- O modelo não está seguindo a instrução de output JSON
- O parse está falhando

**Correção:** 
- Adicionar logs mais detalhados para ver a resposta bruta do Thinker
- Melhorar o prompt para forçar JSON mais definitivamente
- Verificar se `passos` está sendo retornado corretamente

### Problema 3: Redis não salvando
**Local:** `tasks.py` linhas ~1764-1777

O código usa `redis_client` diretamente mas precisa usar `get_redis()`.

**Correção:** Usar `_redis = await get_redis()` em vez de `redis_client` diretamente.

## Locais a modificar:
