# Plano: Thinker como Planejamento Restritivo

## Entendi diferente - Correção

O Thinker **NÃO deve executar**. Ele é apenas **planejador restritivo**.

### Problema atual:
- Orquestrador 2 envia para subagentes
- Subagentes "se perdem" - fazem o que querem, não seguem instruções direito
- Usar modelo grande ajuda, mas cara em tokens

### Solução proposta:
1. Thinker recebe todas as informações do Orquestrador 2 (tools, subagentes, skills)
2. Thinker gera um **plano RESTRITIVO** - passos específicos e detalhados
3. Este plano é enviado ao Orquestrador 2
4. Orquestrador 2 **SÓ pode seguir o plano** - não pode inventar/adaptar
5. Orquestrador 2 distribui para subagentes executarem **APENAS** as tarefas do plano

### Configuração via UI:
- **Checkbox "Execução Restritiva"** na aba Thinker
- Quando ativado: Orquestrador segue **apenas** as instruções do Thinker
- Quando desativado: Comportamento normal (pode adaptar como quiser)

### Como implementar:

#### 1. Frontend: Adicionar campo thinker_restrictive (checkbox)
Em `frontend/src/views/Agents.vue`, aba Thinker:
```
[ ] Ativar Thinker
[ ] Execução Restritiva (orquestrador segue APENAS o plano)
```

#### 2. Backend: Adicionar campo no modelo
Em `backend/app/models/agent.py`:
- `thinker_restrictive` (Boolean, default=False)

#### 3. Backend: Modificar tasks.py
Na área onde o Thinker retorna (~linha 1666-1679):
- Se `thinker_restrictive` está ativado:
  - Injetar no system_prompt com **instrução mais forte** para seguir rigorosamente
  - Adicionar algo como: "**NÃO invente etapas. Execute APENAS os passos abaixo.**"
- Se não: manter comportamento atual

#### 4. Backend: Schema
- Adicionar `thinker_restrictive` no schema do agente
