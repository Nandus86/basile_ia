# Plano: Corrigir carregamento dos campos do Thinker

## Problema:
mesmo ativando o Thinker no "Sistema - General Orch", o log mostra:
```
[Task] 🔍 DEBUG Thinker - is_thinker: False, always_active: False, keywords: [], trigger: [...]
```

O objeto `agent_model` (Agent) não está carregando os campos do banco corretamente - é lazy loading.

## Solução:
Modificar `tasks.py` para buscar o agente diretamente do banco, em vez de usar `agent_config.get("agent_model")`.

O código atual em tasks.py (~linha 1656-1664):
```python
agent_model = agent_config.get("agent_model")
if agent_model:
    is_thinker = getattr(agent_model, 'is_thinker', False)
    ...
```

O problema é que o `agent_model` é gotten do `agent_config`, mas esse objeto pode não ter todos os campos carregados (lazy loading).

## Solução proposta:
Buscar o agente diretamente do banco usando o `agent_id`:
```python
from app.models.agent import Agent
from sqlalchemy import select

# Buscar diretamente do banco
agent_id = agent_config.get("id")  # ou pegar de outra fonte
if agent_id:
    result = await db.execute(select(Agent).where(Agent.id == agent_id))
    agent_model = result.scalar_one_or_none()
```

Isso garantirá que o agente tenha todos os campos carregados corretamente.

## Ação requerida:
Modificar `tasks.py` na seção do Thinker para buscar o agente diretamente do banco.
