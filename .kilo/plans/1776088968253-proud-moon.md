# Plano: Thinker como "Cérebro" integrado ao próprio agente

## Novo Requisito do Usuário:
O Thinker NÃO deve ser um agente separado. Deve ser uma **funcionalidade interna do próprio agente**, como se fosse o "cérebro" que pensa antes de executar.

## Entendendo o Modelo Atual (problema):
- Você cria um agente "Thinker" separado como collaborator
- Usa "thinker_ids" para vincular a outros agentes
- O sistema chama esse Thinker externo como collaborator (causando o problema de ser Disparado do nada)

## Modelo Desejado:
- Thinker é uma **capacidade interna** do agente, não um agente separado
- Quando ativado, o agente "pensa" antes de responder internamente
- Não há chamada para outro agente externo

---

## Respostas do Usuário (confirmadas):

1. **Funcionamento**: Step adicional no prompt (não duas chamadas separadas)
2. **Modelo**: Modelo diferente do agente principal (usar thinker_model)
3. **UI**: Remover "Thinkers Vinculados"

---

## Plano Final de Implementação:

### 1. Backend - Modificar o fluxo de execução

Em `backend/app/worker/tasks.py`:
- Quando o Thinker está ativado, adicionar um step extra no **system_prompt** do agente
- Em vez de chamar um agente Thinker externo, injetar instruções para o próprio agente "pensar" antes de responder
- Usar o `thinker_prompt` configurado pelo usuário
- Usar o `thinker_model` se especificado

### 2. Backend - Simplificar thinker_service.py

- Remover a lógica de chamar agentes Thinker externos
- Manter só a função para gerar o "pensamento" como parte do prompt do agente

### 3. Backend - Remover thinker_ids

- Remover a necessidade de thinker_ids (vincular thinkers)
- O Thinker é agora uma propriedade do próprio agente

### 4. Frontend - Simplificar UI

Remover da aba Thinker:

Manter:

## Fluxo Simplificado:

```
Usuário envia mensagem
        ↓
Agente tem Thinker ativado?
        ↓
Se SIM (sempre ativo ou keyword match):
  → Adicionar instrução no system_prompt para "pensar" antes de responder
  → Instrução usa thinker_prompt (ou padrão do sistema)
  → O próprio agente "pensa" internamente
        ↓
Agente gera resposta (com o pensamento como contexto)
```
