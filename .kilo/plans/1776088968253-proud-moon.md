# Plano: Adicionar campo "thinker_memory_enabled" para ativar memória Redis no Thinker

## Problema:
O usuário quer adicionar um botão/interruptor na interface do Thinker para ativar a memória Redis que armazena a lista de tarefas do Thinker. Quando ativado, o Thinker irá:
1. Criar lista de tarefas
2. Salvar na memória Redis do agente
3. Continuar de onde parou nas próximas interações
4. Limpar memória após completar todas as tarefas

## Campos existentes no modelo (agent.py):
- `is_thinker` - Ativa/desativa o Thinker
- `thinker_prompt` - Prompt customizado
- `thinker_model` - Modelooverride
- `thinker_restrictive` - Execução restritiva
- `thinker_always_active` - Sempre ativo
- `thinker_keywords` - Keywords para ativar

## O que precisa ser feito:

### 1. Backend - Adicionar campo ao modelo (`backend/app/models/agent.py`)
Adicionar novo campo após linha ~131:
```python
thinker_memory_enabled = Column(Boolean, default=True, nullable=False)  # Enable Redis memory for task list
```

### 2. Backend - Adicionar campo aos schemas (`backend/app/schemas/agent.py`)
- `AgentCreate.thinker_memory_enabled`
- `AgentUpdate.thinker_memory_enabled`  
- `AgentResponse.thinker_memory_enabled`

### 3. Backend - Migration (`backend/run_migration_thinker.py`)
Adicionar verificação e adição da coluna:
```python
# 8. Check and add thinker_memory_enabled column
check_memory = text("""
    SELECT column_name 
    FROM information_schema.columns 
    WHERE table_name='agents' AND column_name='thinker_memory_enabled';
""")

if exists:
    print("Column 'thinker_memory_enabled' already exists.")

    await conn.execute(text("ALTER TABLE agents ADD COLUMN thinker_memory_enabled BOOLEAN DEFAULT TRUE NOT NULL;"))
    print("✅ Column 'thinker_memory_enabled' added")
```

### 4. Frontend - Adicionar switch na UI (`frontend/src/views/Agents.vue`)
No tab "Thinker", adicionar novo switch após "Execução Restritiva":
```vue

  label="Memória Ativa"

</p>
```

### 5. Backend - Modificar lógica do Thinker (`tasks.py` e `agent_orchestrator.py`)
Modificar para verificar `thinker_memory_enabled` antes de:
- Salvar lista de tarefas na memória Redis
- Continuar de tarefas anteriores
- Limpar memória após completar

Se `thinker_memory_enabled=False`, o Thinker funciona de forma simples (apenas injeta instrução, sem lista de tarefas persistida).
