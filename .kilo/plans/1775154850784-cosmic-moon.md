# Plano de Melhoria da Área de Inserção de Informações

## Objetivo
Ampliar a área de inserção de informações nas telas de edição (Agentes, MCPs, Skills, Bases de Informação), tornando os campos de texto maiores e adicionando um botão para expandir a tela durante o preenchimento.

## Problema Atual
- Os diálogos de edição têm tamanho limitado (`max-width="900"` ou `max-width="800"`)
- Campos de texto como `v-textarea` têm poucas linhas (`rows="2"`, `rows="4"`, etc.)
- Não há opção de expandir a tela para melhor visualização durante a edição

## Soluções Propostas

### 1. Aumentar o Tamanho dos Diálogos
- Alterar `max-width` dos diálogos de `900` para `1200` ou `1400`
- Isso dará mais espaço horizontal para os campos

### 2. Ampliar os Campos de Texto
- Aumentar o número de `rows` em todos os `v-textarea`:
  - `system_prompt`: de `rows="5"` para `rows="8"`
  - `intent`: de `rows="4"` para `rows="6"`
  - `content_md`: de `rows="12"` para `rows="15"`
  - `description`: de `rows="2"` para `rows="4"`
  - Campos JSON (schemas, headers, body): de `rows="4-5"` para `rows="6-8"`

### 3. Adicionar Botão de Expandir Tela
- Incluir um botão `v-btn` com ícone `mdi-fullscreen` no cabeçalho do diálogo
- Ao clicar, alternar uma variável `isExpanded` entre `true`/`false`
- Quando `isExpanded` for `true`:
  - Alterar `max-width` do diálogo para `95vw` (95% da largura da viewport)
  - Opcionalmente adicionar `fullscreen` ou `max-height="95vh"`
- Ícone muda para `mdi-fullscreen-exit` quando expandido

### 4. Implementação por Arquivo

#### Agents.vue
- Diálogo principal: `max-width="900"` → `max-width="1200"`
- Adicionar botão de expandir no `v-card-title` do diálogo
- Aumentar `rows` em:
  - `system_prompt`: `rows="8"`
  - `description`: `rows="4"`
  - Campos JSON nas abas de Input/Output

#### MCP.vue
- Diálogo principal: `max-width="900"` → `max-width="1200"`
- Aumentar `rows` em:
  - `description`: `rows="4"`
  - `endpoint`: manter `auto-grow` mas com `rows="3"` mínimo
  - `headersJson`: `rows="8"`
  - `bodyJson`: `rows="8"`
  - `queryTemplateJson`: `rows="4"`
  - `responseMappingJson`: `rows="6"`

#### Skills.vue
- Diálogo principal: `max-width="900"` → `max-width="1200"`
- Aumentar `rows` em:
  - `intent`: `rows="6"`
  - `content_md`: `rows="15"`

#### InformationBases.vue
- Diálogo principal: `max-width="800"` → `max-width="1000"`
- Aumentar `rows` em:
  - `content_schema`: `rows="8"`
  - `metadata_schema`: `rows="6"`
  - `correlation_schema`: `rows="4"`

### 5. Componente Reutilizável para Botão de Expandir
Criar um componente opcional ou função helper que:
- Recebe referência ao diálogo
- Gerencia estado de expansão
- Alterna classes CSS ou props dinamicamente

### 6. Melhorias Adicionais (Opcional)
- Adicionar `auto-grow` em todos os `v-textarea` para expandirem conforme conteúdo
- Usar `density="comfortable"` para mais espaçamento interno
- Adicionar hint persistente sobre atalhos de teclado (ex: `Ctrl+Enter` para expandir)
