# Plano: Fluxo Sequencial para Skills + HITL

## Conceito

O sistema permite que skills definam fluxos de execução sequenciais com pauses obrigatórias em pontos específicos:

- `<<FLOW_START>>` ... `<<FLOW_END>>` - Delimita uma etapa do fluxo
- `{{ $HITL }}` - Pausa obrigatória, devolve para o usuário antes de continuar

### Como Funciona

1. **Detecção**: Sistema detecta qual skill é necessária baseado na mensagem do usuário
2. **Extração**: Extrai TODOS os fluxos (etapas) da skill identificada
3. **Injeção**: Injeta todos os fluxos no contexto com instrução de execução sequencial
4. **Execução**: 
   - IA executa Etapa 1
   - Se tem `{{ $HITL }}`, pausa e aguarda resposta do usuário
   - Com resposta, executa Etapa 2
   - Repete até completar todas as etapas

### Arquitetura

#### 1. skill_detector.py - Extrair Todas as Etapas

```python
def extract_all_flows(skill: Skill) -> List[Dict]:
    """
    Extrai todas as etapas de fluxo de uma skill.
    Cada etapa tem: flow_content, has_hitl, etapa_number
    """
    content = skill.content_md or ""
    flows = []
    
    # Encontrar todos os blocos <<FLOW_START>>...<<FLOW_END>>
    pattern = r'<<FLOW_START>>(.*?)<<FLOW_END>>'
    matches = re.finditer(pattern, content, re.DOTALL)
    
    for i, match in enumerate(matches, 1):
        flow_content = match.group(1).strip()
        
        # Verificar se tem $HITL APÓS este bloco de flow
        # (pode estar em outra linha depois do <<FLOW_END>>)
        end_pos = match.end()
        remaining = content[end_pos:end_pos+500]  # Verificar próximo texto
        has_hitl = bool(re.search(r'\{\{\s*\$HITL\s*\}\}', remaining))
        
        flows.append({
            "etapa": i,
            "flow": flow_content,
            "has_hitl": has_hitl
        })
    
    return flows
```

## Decisões Confirmadas

1. **Nome**: "Skill Router"
2. **Escopo**: Analisa todas as mensagens
3. **Nome dos marcadores**: `<<FLOW_START>>`/`<<FLOW_END>>`
4. **HITL**: `{{ $HITL }}` após cada bloco de fluxo
5. **Sem fluxos na skill**: Não injetar nada
6. **Skill sem detecção**: Não injetar nada
7. **Conexão gerador**: Gerador cria skills com padrão de etapas

## Arquitetura do Skill Router

### 1. Como funciona

1. **Mensagem do usuário** chega ao orchestrator
2. **Skill Router** analisa a mensagem contra as capabilities disponíveis
3. **Se identificar skill**: injeta o fluxo da skill no contexto do agente executor
4. **Se não identificar**: continua o fluxo normal sem skill

### 2. Capabilities Resumo

O Skill Router recebe apenas as descrições resumidas das capabilities (headers ## com resumo <<>>), não as skills completas:

```
- register_cell_attendance_list: Inserir lista de presença de células
- insert_member: Inserir novo membro na célula
- delete_member: Remover membro da célula
- list_cells: Listar todas as células
- send_whatsapp_message: Enviar mensagem via WhatsApp
```

### 3. Resposta do Router

- Se encontrar match: retorna o nome da capability identificada
- Se não encontrar: retorna "NENHUMA"

### 4. Fluxo no Graph

```python
# Antes de gerar resposta
skill_route = await skill_router.analyze(user_message)

if skill_route and skill_route.get("capability"):
    # Injetar fluxo da skill no contexto
    inject_skill_flow(skill_route["skill"])
```

## Implementação

### 1. skill_router.py - Novo módulo

```python
"""
Skill Router - Analisa mensagens e determina qual skill usar
"""
from typing import Optional, Dict
from langchain_openai import ChatOpenAI
from langchain.schema import SystemMessage, HumanMessage

class SkillRouter:
    def __init__(self, model_name: str = "gpt-4o-mini"):
        self.llm = ChatOpenAI(model=model_name, temperature=0)
    
    def _build_capabilities_text(self, capabilities: list) -> str:
        lines = []
        for cap in capabilities:
            header = cap.get("header", "")
            desc = cap.get("description", "")
            lines.append(f"- {header}: {desc}")
        return "\n".join(lines)
    
    async def analyze(self, user_message: str, skills: list) -> Optional[Dict]:
        from app.schemas.skill import get_skills_capabilities_summary
        
        all_capabilities = []
        for skill in skills:
            if skill.is_active:
                caps = get_skills_capabilities_summary(skill)
                for cap in caps:
                    all_capabilities.append({
                        "header": cap["header"],
                        "description": cap["description"],
                        "skill_name": skill.name
                    })
        
        if not all_capabilities:
            return None
        
        capabilities_text = self._build_capabilities_text(all_capabilities)
        
        system_prompt = f"""Você é um roteador de intents. Analise a mensagem do usuário e determine:

1. O usuário está fazendo uma solicitação que requer uma skill?
2. Se sim, qual capability/skill deve ser usada?

## Capabilities Disponíveis:
{capabilities_text}

## Regras:
- Responda APENAS com o nome da capability identificada
- Se nenhuma skill for necessária, retorne "NENHUMA"
- Use o nome EXATO da capability conforme listado acima

Responda apenas com o nome da capability ou 'NENHUMA'."""

        messages = [
            SystemMessage(content=system_prompt),
            HumanMessage(content=user_message)
        ]
        
        response = await self.llm.ainvoke(messages)
        capability_name = response.content.strip()
        
        if capability_name == "NENHUMA":
            return None
        
        for cap in all_capabilities:
            if cap["header"] == capability_name:
                for skill in skills:
                    if skill.name == cap["skill_name"]:
                        return {"skill": skill, "capability": cap}
        
        return None
```

## Feature Adicional: Skill Sempre Ativa

### Problema

Algumas skills devem ser sempre consideradas pelo Skill Router, independente da detecção por keywords ou similar.

### Solução Proposta

Adicionar um campo `always_active` na skill (ou via frontmatter YAML na skill) que força o router a sempre considerar essa skill:

```yaml
---
name: Nome da Skill
description: Descrição
always_active: true
---
```

### Modificações Necesárias

1. **Skill Model**: Adicionar campo `always_active: bool` 
2. **Skill Router**: Verificar skills com `always_active=True` e incluir suas capabilities independente de análise
3. **Frontend**: Adicionar checkbox "Sempre Ativa" na edição da skill
